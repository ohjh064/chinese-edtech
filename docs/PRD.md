# 중국어 어휘 수행평가 플랫폼 PRD

> **작업 제목(가칭):** 핀마스터(PinMaster) — 중국어 어휘 활용하기 수행평가 시스템
> **문서 버전:** v0.1 (초안)
> **작성 목적:** 교사가 만든 평가기준을 그대로 채점 엔진으로 구현하여, 학생 연습 → 수행평가 응시 → 자동채점 → 교사 검수 → 피드백 → 학습관리까지 한 흐름으로 처리하는 웹 플랫폼

---

## 1. 제품 개요

### 1.1 한 줄 정의
중국어 어휘(병음·성조·의미·문장) 수행평가를, **학생이 집에서 연습하고**, **시스템이 오류 개수를 세어 자동채점하고**, **교사가 검수·피드백**하는 웹 기반 평가 플랫폼.

### 1.2 배경
- 현행 어휘 수행평가는 종이 답안 → 교사 수기 채점 → 오류 개수 카운트 → 척도 매칭의 반복 작업이다.
- 병음 표기 오류, 성조 오류, 의미 오류, 어법 오류를 **각각 따로** 세어야 해서 채점 부담이 크다.
- 학생은 평가 전 연습 환경이 마땅치 않고, 본인이 어디서 틀리는지(성모인지, 운모인지, 성조인지) 즉시 피드백을 못 받는다.
- 본 시스템은 **"오류 개수 → 점수"가 결정론적으로 떨어지는 평가기준의 특성**을 활용해 병음·성조 채점을 완전 자동화하고, 의미·문장은 AI 보조 + 교사 확정으로 처리한다.

### 1.3 평가기준 요약(업로드 기준표 기반)

영역: **가. 어휘 활용하기 (전체 평가의 30%)** / 관련단원: 중국어 발음, 1. 你好, 2. 我是韩国人

| 평가 요소 | 배점 | 채점 대상 | 자동화 가능성 |
|---|---|---|---|
| 병음표기정확성 | 25 | 성모·운모 표기 오류 | ◎ 완전 자동 |
| 성조변별정확성 | 25 | 성조 표기 오류 | ◎ 완전 자동 |
| 의미변별정확성 | 25 | 단어 의미 작성 오류 | ○ AI+교사 보조 |
| 오류판단정확성 | 25 | 단어 활용 문장 어법 오류 | △ AI 보조 + 교사 확정 |
| **합계** | **100** | | |

**오류 개수 → 점수 척도(4개 영역 공통):**

| 오류 개수 | 점수 |
|---|---|
| 0개 | 25 |
| 1~2개 | 20 |
| 3~4개 | 15 |
| 5~6개 | 10 |
| 7개 이상 | 5 |

- 기본점수(수행평가 참여 학생의 최하점): **20**
- 장기 미인정 결석 등 추가평가 불가 미응시자: **10** (기본점수의 50%)

> ⚠️ **설계 메모:** "오류판단정확성" 영역은 기준표 명칭은 *오류 판단*이지만, 채점 설명은 *"주어진 단어를 활용하여 어법에 맞게 문장을 작성"*입니다. 본 PRD는 **단어를 활용한 작문 → 어법 오류 카운트** 방식으로 설계하되, 교사가 과제 유형을 *작문형 / 오류 찾기형* 중에서 고를 수 있게 옵션화합니다(§5.5, §15 참조).

## 2. 문제 정의 & 목표

### 2.1 문제 정의 (요청 사항)
1. 교사가 학생의 평가 과정을 **피드백**할 수 있다.
2. **채점이 용이**하다(오류 카운트·척도 매칭 자동화).
3. **학습을 관리**할 수 있다(반·개인 단위 진척·취약점 추적).
4. 학생이 **집에서도 연습**할 수 있다.

### 2.2 목표(Goals)
- 병음·성조 채점을 **100% 자동화**해 교사의 수기 카운트 작업을 제거한다.
- 학생이 본 평가 전에 **무제한 연습 + 즉시 피드백**을 받을 수 있게 한다.
- 교사가 한 화면에서 **반 전체 답안을 검수·확정·일괄 점수 산출**할 수 있게 한다.
- 최종 점수를 **NEIS 입력용 형식(엑셀)으로 내보내기** 가능하게 한다.

### 2.3 성공 지표(Success Metrics)
| 지표 | 측정 방법 | 목표 |
|---|---|---|
| 채점 시간 절감 | 종이 채점 대비 한 반 채점 소요시간 | 70% 이상 절감 |
| 자동채점 정확도(병음·성조) | 표본 답안 수기 대조 일치율 | 99% 이상 |
| 학생 연습 참여율 | 평가 전 1회 이상 연습한 학생 비율 | 80% 이상 |
| 교사 검수 부담 | 의미·문장 영역 교사 수정 비율 | AI 1차 채점 후 수정 30% 이하 |

### 2.4 범위 밖(Out of Scope) — 최소 1차 버전 기준
- 듣기/말하기 평가(발음 녹음 채점)
- 나머지 70% 영역(어휘 활용하기 외 다른 수행평가 영역)
- NEIS 직접 연동(우선은 엑셀 내보내기로 대체)
- 학부모 계정

## 5. 평가·채점 엔진 (제품의 심장 ★)

이 섹션이 기준표를 실제 코드 로직으로 옮긴 부분입니다. 영역별로 **무엇을 오류 1개로 셀지** 정의가 핵심입니다.

### 5.0 공통: 오류 개수 → 점수 매핑 함수

```
function mapErrorsToScore(errorCount):
    if errorCount == 0: return 25
    if errorCount <= 2: return 20    // 1~2개
    if errorCount <= 4: return 15    // 3~4개
    if errorCount <= 6: return 10    // 5~6개
    return 5                          // 7개 이상
```

### 5.1 병음표기정확성(25) — 완전 자동

**오류 정의:** 한어병음 정사법 기준, 단어의 **성모(initial)·운모(final)** 표기 오류. **성조는 제외**(성조는 §5.2에서 따로 카운트).

**채점 알고리즘:**
1. 학생 답안에서 성조(부호 또는 숫자)를 분리·제거 → 순수 로마자 병음만 남김
2. 음절 단위로 분해, 각 음절을 (성모, 운모)로 파싱
3. 정답키의 (성모, 운모)와 대조
4. **성모 1개 불일치 = 오류 1개, 운모 1개 불일치 = 오류 1개**

예시: 你好 정답 `nǐ hǎo` → 성모 [n, h], 운모 [i, ao]
- 학생이 `lǐ hǎo` 입력 → 성모 n→l 오류 **1개**
- 학생이 `nǐ hǒu` 입력 → 운모 ao→ou 오류 **1개**

```
function gradePinyin(studentAnswers, answerKey):
    errors = 0
    for each word:
        student = stripTones(student.pinyin)   # 부호/숫자 제거
        for each syllable index i:
            if student.initial[i] != key.initial[i]: errors += 1
            if student.final[i]   != key.final[i]:   errors += 1
    return { errors, score: mapErrorsToScore(errors) }
```

> 🔧 **설정값(교사 선택):** 오류 카운트 단위를 *(a) 성모/운모 각각 1개* / *(b) 음절당 1개* / *(c) 단어당 1개* 중 선택 가능하게. 기본값은 **(a)** — 기준표 문구("성모와 운모의 표기 오류")에 가장 충실.

### 5.2 성조변별정확성(25) — 완전 자동

**오류 정의:** 제시 단어의 성조 표기 오류. **음절별 성조 1개 불일치 = 오류 1개.**

**채점 알고리즘:**
1. 학생 답안에서 음절별 성조(1·2·3·4성, 경성=5 또는 0) 추출
2. 정답키 성조와 음절 단위 대조

예시: 你好 정답 성조 [3, 3]
- 학생이 [2, 3] 입력 → 첫 음절 성조 오류 **1개**

```
function gradeTones(studentAnswers, answerKey):
    errors = 0
    for each word:
        for each syllable index i:
            if student.tone[i] != key.tone[i]: errors += 1
    return { errors, score: mapErrorsToScore(errors) }
```

> 💡 병음·성조를 따로 채점하므로, **학생 입력 단계에서 로마자와 성조를 분리 저장**하는 게 채점에 유리합니다(§8 입력 UX와 직결).

### 5.3 의미변별정확성(25) — AI + 교사 보조

**오류 정의:** 제시 단어의 한국어 의미 작성 오류. **단어 1개의 의미가 틀리면 오류 1개.**

**채점 알고리즘(3단계, 비용 효율 순서):**
1. **정확 일치 / 정규화 매칭(비용 0):** 교사가 등록한 허용 정답 목록과 비교(띄어쓰기·조사·동의어 표준화).
   - 예: 你好 → 허용정답 [`안녕`, `안녕하세요`, `여보세요(전화)`]
2. **사전형 동의어 매칭(비용 0):** 내장 동의어/유사어 테이블로 1차 통과.
3. **AI 판정(비용 발생, fallback만):** 1·2단계에서 못 맞춘 답안만 모아 한 번에 LLM에 "이 답이 정답으로 인정 가능한가" 질의 → 인정 불가만 오류로 카운트.

```
function gradeMeaning(studentAnswers, answerKey):
    uncertain = []
    errors = 0
    for each word:
        if matchExact(student.meaning, key.acceptable): continue        # 통과
        if matchSynonym(student.meaning, key.acceptable): continue      # 통과
        uncertain.push(word)                                            # AI로 위임
    aiResults = batchAIJudge(uncertain)    # 단 1회 호출로 묶음 처리
    errors += countRejected(aiResults)
    return { errors, score: mapErrorsToScore(errors), needsReview: uncertain }
```

> 💰 **비용 효율 원칙:** 정확 일치로 거를 수 있는 답안은 절대 AI를 부르지 않는다. AI는 *애매한 답안만, 묶어서, 한 번에* 호출한다.

### 5.4 오류판단정확성(25) — AI 보조 + 교사 확정

**오류 정의:** 주어진 단어를 활용한 문장의 어법 오류. **어법 오류 1개 = 오류 1개.**

**채점 알고리즘:**
1. 학생이 작성한 모든 문장을 수집
2. LLM에 **묶음(batch)으로** 어법 검사 요청 → 문장별 오류 개수 + 오류 위치/사유 반환
3. 합산 → 척도 매칭
4. **교사 확정 필수**(어법 판정은 자동값을 제안값으로만 사용, 교사가 최종 수정)

```
function gradeSentences(studentSentences):
    result = batchAIGrammarCheck(studentSentences)   # 학생 전체 문장 1회 호출
    errors = sum(result.errorCountPerSentence)
    return { errors, score: mapErrorsToScore(errors),
             aiSuggested: true, requiresTeacherConfirm: true }
```

> 🔧 **과제 유형 옵션(§1.3 메모 연동):**
> - **작문형:** 학생이 단어로 문장 작성 → AI가 어법 오류 카운트(위 알고리즘)
> - **오류 찾기형:** 교사가 오류 포함 문장 제시 → 학생이 오류를 찾아 수정 → 정답 대조(완전 자동 가능)

### 5.5 최종 점수 산출

```
total = pinyinScore + toneScore + meaningScore + sentenceScore   # 최대 100

# 기본점수 하한 적용
if 학생.응시 == true:
    final = max(total, 20)           # 참여자 최하점 20
else if 학생.장기미인정결석:
    final = 10                       # 기본점수의 50%
```

## 6. 학생 학습 흐름

```
[연습 모드] ──반복──> [자가 진단] ──> [본 평가 응시] ──> [제출]
     │                                                      │
   즉시 자동 피드백                                    교사 검수·확정
   (병음/성조 정오, 틀린 부분 하이라이트)                      │
                                                      [영역별 피드백 수령]
```

### 6.1 연습 모드(집에서)
- 같은 과제 형식, **무제한 시도**, 정답 즉시 공개 옵션(교사 설정).
- 병음·성조는 입력 즉시 자동 채점 → **어느 음절의 성모/운모/성조가 틀렸는지 하이라이트**.
- 의미·문장은 AI 코멘트(연습용은 가벼운 힌트 수준).
- 연습 기록 누적 → 약한 단어 자동 추출 → "복습 추천".

### 6.2 평가 응시 모드(본 수행평가)
- 교사가 지정한 시간 안에 **1회 응시**(또는 교사 허용 횟수).
- 응시 중 정답 비공개, 타이머 표시, 제출 후 수정 불가.
- 제출 즉시 병음·성조 자동채점 / 의미·문장은 교사 검수 큐로 이동.

## 7. 주요 화면 구성

### 교사
- **대시보드:** 진행 중 평가, 미검수 답안 수, 반별 평균.
- **평가 출제 화면:** 단어 입력 그리드(한자 / 정답 병음 / 성조 / 허용 의미 / 예문) — 한자 입력 시 병음·성조 **자동 추천**(교사가 확인·수정).
- **검수 화면:** 학생별·문항별 답안 + AI 제안 점수 → 인라인 수정 → 확정. 반 전체 일괄 보기/필터(예: "성조 틀린 학생만").
- **분석 화면:** 오류 히트맵(단어 × 오류유형), 개인 리포트, 엑셀 내보내기.

### 학생
- **내 평가 목록:** 연습 가능 / 응시 대기 / 완료.
- **응시·연습 화면:** 단어별 입력 칸(병음 + 성조 + 의미 + 문장), §8 입력기 탑재.
- **피드백 화면:** 영역별 점수, 틀린 부분 하이라이트, AI 코멘트, 복습 추천.

## 8. 핀인 입력 UX (특별 주의 ★)

학생이 키보드로 성조 부호(nǐ hǎo)를 입력하기 어렵다는 점이 **제품 채택의 가장 큰 장벽**입니다. 채점 로직이 병음과 성조를 분리해 다루므로, 입력 단계에서부터 둘을 분리 저장하는 설계를 권장합니다.

**권장안 — 숫자 성조 입력 + 실시간 변환:**
- 학생은 `ni3 hao3`처럼 **로마자 + 숫자(1~4, 경성 0/5)** 로 입력.
- 시스템이 즉시 `nǐ hǎo`로 변환 미리보기 표시.
- 저장 시 **로마자(병음) / 숫자(성조)를 분리 저장** → 채점 그대로 활용.
- 장점: 어떤 키보드에서도 입력 가능, 모바일 친화적, 병음·성조 분리 채점에 최적.

**대안 검토:**
| 방식 | 장점 | 단점 |
|---|---|---|
| 숫자 성조(`ni3`) | 입력 쉬움, 분리 저장 용이 | 학생이 규칙 1회 학습 필요 |
| 성조 버튼 팔레트 | 직관적 | 클릭 많아 느림 |
| 음절별 분리 입력칸 | 채점 명확 | 화면 복잡, 입력 번거로움 |

> 권장: **숫자 성조 입력**을 기본으로, 처음 사용 시 1회 튜토리얼 + 성조 버튼 보조 제공.

| 내보내기 | 서버 엑셀 생성 | NEIS 입력용 |

**핵심 아키텍처 포인트:**
- 병음·성조 채점은 **클라이언트/서버 로컬 로직**으로 처리(AI 호출 0 → 비용 0, 즉시 응답).

- 한자 입력 시 라이브러리로 병음·성조 **자동 추천** → 교사 정답키 입력 부담 대폭 절감.

## 11. AI 활용 & 비용 효율화 원칙

1. **결정론적 채점은 AI 금지:** 병음·성조는 100% 로컬 로직. (비용 0, 지연 0)
2. **AI는 fallback + batch:** 의미는 1차 매칭 실패분만, 문장은 학생 전체 문장을 **한 번에** 호출.
3. **공통 피드백 캐싱:** 같은 단어에서 같은 오류 패턴이 반복되면 피드백 문구 재사용.
4. **연습 vs 평가 호출 분리:** 연습 모드의 가벼운 힌트는 규칙 기반/캐시 우선, AI는 본 평가 검수에 집중.

## 13. MVP 범위 & 개발 로드맵

### MVP(1차) — "채점 자동화"부터
- 교사 평가 출제(단어 세트 + 정답키, 한자→병음 자동추천)
- 학생 응시(숫자 성조 입력기)
- **병음·성조 자동채점 + 척도 매칭** ← 가장 임팩트 큰 부분
- 의미: 정확 일치 매칭(AI 없이)
- 문장: 교사 수기 입력(자동채점 보류)
- 점수 산출 + 엑셀 내보내기

### 2차 — "AI 보조 + 연습"
- 의미 AI fallback 판정
- 문장 AI 어법 검사(교사 확정형)
- 연습 모드 + 즉시 피드백 + 약점 복습 추천

### 3차 — "학습관리 고도화"
- 반/개인 분석 대시보드, 오류 히트맵
- 과제 유형 옵션(오류 찾기형)
- 누적 학습 이력 리포트

## 15. 열린 결정사항(JH 확정 필요)

1. **오류판단 영역 해석:** 작문형(단어 활용 작문 → 어법 오류 카운트) vs 오류 찾기형(오류 문장 제시 → 수정) — 어느 쪽이 실제 출제 의도인가요? (둘 다 옵션 제공 예정이나 기본값 결정 필요) → 사용자가 선택할 수 있게.
2. **병음 오류 카운트 단위:** 성모/운모 각각(권장) vs 음절당 vs 단어당 — 학교 채점 관행에 맞춰 기본값을 정해야 합니다. → 사용자가 선택할 수 있게.
3. **의미 오류 기준:** 한 단어에 부분정답(예: 뜻은 맞지만 품사·뉘앙스 어긋남)을 0.5개로 셀지, 1개로 셀지. → 사용자가 선택할 수 있게.
4. **연습 모드 정답 공개 범위:** 연습 중 정답을 바로 보여줄지, 시도 후에만 보여줄지. → 사용자가 선택할 수 있게.
5. **응시 통제 수준:** 단순 타이머인지, 탭 이탈 감지 등 부정행위 방지까지 필요한지. → 사용자가 선택할 수 있게.
6. **계정 생성 방식:** 교사가 학생 계정 일괄 발급. → 사용자가 선택할 수 있게.
